## 🎯 关键洞察

### 技术架构核心发现

**三层分离架构是最优解**:
```
用户 → [对话层: LLM] → [知识层: RAG] → [计算层: 规则引擎]
```

**为什么？**
- LLM不需要"学会算命"，只需要"会表达"
- 规则引擎保证命理计算的准确性
- RAG提供知识依据，避免LLM幻觉
- 各层可以独立优化

---

### 一致性问题的解决方案

**核心思路**: 统一用户画像 + 多视角表达

```
底层: 统一的性格特征、运势趋势
表达: 不同系统的语言风格
```

**举例**:
- 八字: "你五行缺木，性格内敛"
- 星座: "你太阳处女，注重细节"
- 塔罗: "你抽到隐士牌，喜欢独处"
→ 底层一致: 内向、理性、注重细节

## 🚀 本次迭代的决策

### 1. 技术架构确定

**方案**: 三层分离架构
- 对话层: GPT-4 (负责温暖表达)
- 知识层: ChromaDB + RAG (检索命理知识)
- 计算层: Python规则引擎 (精确计算命盘)

**MVP技术栈**:
- 后端: FastAPI + PostgreSQL + ChromaDB
- AI: GPT-4 API (MVP阶段)
- 前端: React (网页版，快速开发)

---

### 2. MVP范围再收窄

**只做"八字大师" persona**:
- ✅ 用户注册/登录
- ✅ 八字排盘计算
- ✅ AI对话 (基于RAG)
- ✅ 付费墙

**暂不做**:
- ❌ 星座、塔罗 (V1.5再做)
- ❌ 复杂UI
- ❌ 社区功能

**原因**: 2人兼职，时间有限，先验证核心假设

---

### 3. 知识库策略

**数据来源**:
- 命理经典 (易经、滴天髓等)
- 现代命理书籍
- 真人案例 (脱敏)

**组织方式**:
- 向量化存储 (ChromaDB)
- 分层: 理论 → 案例 → 解读模板
- 版本控制 (持续迭代)
