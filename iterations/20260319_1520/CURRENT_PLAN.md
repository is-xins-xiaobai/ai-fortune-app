## 🔧 技术架构 (已确定)

### 三层分离架构

```
用户 → [对话层: LLM] → [知识层: RAG] → [计算层: 规则引擎]
```

**各层职责**:
1. **对话层** (GPT-4): 温暖表达，共情能力
2. **知识层** (ChromaDB): 存储和检索命理知识
3. **计算层** (Python): 精确的命盘计算

**为什么这样设计？**
- ✅ LLM不需要"学会算命"，只需要"会表达"
- ✅ 规则引擎保证计算准确性
- ✅ RAG提供知识依据，避免幻觉
- ✅ 各层可以独立优化

## 🎯 一致性设计 (已解决)

### 核心问题
如何保持八字/星座/塔罗之间的一致性？

### 解决方案: 统一底层 + 多元表达

```
统一用户画像 → 各系统计算 → 冲突检测 → 多视角表达
```

### 三层保障

1. **统一用户画像层**
   - 存储核心性格特征
   - 存储运势趋势
   - 所有系统基于同一画像

2. **命理计算层**
   - 各系统独立计算
   - 结果转换为统一格式
   - 便于比较和冲突检测

3. **多视角表达层**
   - 不同"大师" persona
   - 用各自系统语言风格表达
   - 但底层结论一致

### MVP建议

**先只做八字**，验证核心假设后再扩展：
- Phase 1: 八字大师 (MVP)
- Phase 2: 增加星座 (处理两系统一致性)
- Phase 3: 增加塔罗 (完善多系统)
