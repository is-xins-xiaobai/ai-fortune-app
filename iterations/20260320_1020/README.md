## 🗄️ 数据库选型

### 数据类型分析

| 数据类型 | 特点 | 存储需求 |
|---------|------|---------|
| **用户档案** | 结构化，小数据量 | 关系型数据库 |
| **对话历史** | 大量文本，需检索 | 关系型+向量数据库 |
| **记忆数据** | 快速读取，临时性 | 内存数据库 |
| **统计数据** | 时序数据 | 关系型数据库 |

### 推荐方案: PostgreSQL + Redis

#### PostgreSQL (主数据库)

**选择理由**:
- ✅ 成熟稳定，免费开源
- ✅ 支持JSONB (存储灵活的记忆数据)
- ✅ 全文搜索 (检索对话历史)
- ✅ 托管服务多 (RDS, Supabase等)

**表结构设计**:
```sql
-- 用户表
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    email VARCHAR(255) UNIQUE,
    password_hash VARCHAR(255),
    birth_date DATE,
    birth_time TIME,
    birth_location VARCHAR(255),
    created_at TIMESTAMP DEFAULT NOW(),
    subscription_status VARCHAR(50) DEFAULT 'free',
    subscription_end_date TIMESTAMP
);

-- 对话表
CREATE TABLE conversations (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id),
    title VARCHAR(255),
    summary TEXT,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- 消息表
CREATE TABLE messages (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    conversation_id UUID REFERENCES conversations(id),
    role VARCHAR(50), -- 'user' or 'assistant'
    content TEXT,
    tokens_used INTEGER,
    created_at TIMESTAMP DEFAULT NOW()
);

-- 记忆表 (JSONB存储灵活数据)
CREATE TABLE memories (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id),
    memory_type VARCHAR(50), -- 'fact', 'preference', 'event'
    content JSONB,
    importance_score FLOAT,
    created_at TIMESTAMP DEFAULT NOW(),
    last_accessed TIMESTAMP DEFAULT NOW()
);
```

#### Redis (缓存层)

**用途**:
1. **会话缓存** - 活跃用户的对话上下文
2. **记忆缓存** - 快速读取用户记忆
3. **限流控制** - API调用频率控制
4. **消息队列** - 异步任务处理

**数据结构**:
```redis
# 用户会话缓存 (TTL: 1小时)
user:session:{user_id} -> {conversation_context}

# 用户记忆缓存 (TTL: 24小时)
user:memory:{user_id} -> [memory_items]

# 对话历史缓存 (最近10条)
conversation:history:{conversation_id} -> [messages]

# API限流计数
rate_limit:{user_id}:{hour} -> count
```

### 备选: Supabase (BaaS)

**如果选择无服务器方案**:
- ✅ 托管PostgreSQL
- ✅ 内置认证
- ✅ 实时订阅
- ✅ 免费额度 generous

**适合**: 想快速上线，不想管理服务器

### 成本优化建议
