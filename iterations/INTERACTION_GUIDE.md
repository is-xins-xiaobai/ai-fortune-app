# 互动指南 - 如何参与企划迭代

**目的**: 让你可以随时参与讨论、提出要求、给出反馈

---

## 💬 如何与我互动？

### 1. 查看当前方案

```bash
# 查看最新迭代概要
cat iterations/current/README.md

# 查看详细方案
cat iterations/current/CURRENT_PLAN.md

# 查看所有文件
ls -la iterations/current/
```

---

### 2. 提出反馈/要求

直接在对话中告诉我，我会：
1. 理解你的意图
2. 在下次迭代中融入你的想法
3. 在审查报告中说明如何响应

**示例**:
```
你: "我觉得定价太高了，能不能降到¥19/月？"
我: "好的，我会在下次迭代中重新评估定价策略，
    并分析¥19 vs ¥29的优劣势。"
```

---

### 3. 触发即时迭代

如果你想立即看到新方案，可以说：

```
"立即进行一次迭代"
"我有新想法，重新推演一次"
"基于我刚才的反馈，更新方案"
```

---

### 4. 查看历史迭代

```bash
# 查看所有历史迭代
ls -la iterations/

# 查看某次迭代的内容
ls -la iterations/20260319_1420/
cat iterations/20260319_1420/README.md
```

---

### 5. 暂停/恢复自动迭代

```
"暂停自动迭代" - 停止定时任务
"继续自动迭代" - 恢复定时任务
```

---

## 📋 互动命令速查

| 你说 | 我会 |
|------|------|
| "立即迭代" | 马上进行下一次迭代 |
| "我有反馈..." | 在下次迭代中融入 |
| "查看当前方案" | 展示最新的方案概要 |
| "暂停迭代" | 停止自动迭代 |
| "继续迭代" | 恢复自动迭代 |
| "重点关注X" | 下次迭代重点优化X方向 |

---

## 🎯 迭代节奏

### 自动模式 (默认)
- 每小时20分自动迭代
- 你可以随时查看最新方案
- 你的反馈会在下次迭代中体现

### 手动模式
说"暂停自动迭代"：
- 我会停止定时任务
- 等你明确指令再迭代
- 适合你想深度参与每次决策

---

## 📊 每次迭代的产出

每次迭代会在 `iterations/YYYYMMDD_HHMM/` 生成：

```
20260319_1420/
├── README.md              # 本次迭代概要
├── CURRENT_PLAN.md        # 当前主方案
├── VALUE_ANALYSIS.md      # 价值分析
├── REVIEW_REPORT.md       # 审查报告
├── USER_FEEDBACK.md       # 你的反馈记录
└── NEXT_ACTIONS.md        # 下一步行动
```

## 🎪 示例对话

### 场景1: 质疑定价

```
你: "¥29/月会不会太贵？我担心转化率低"

我: "好问题！让我在下次迭代中：
    1. 分析¥19 vs ¥29 vs ¥39的不同定价策略
    2. 对比竞品定价
    3. 计算不同价格下的收入预测
    4. 给出A/B测试方案
    
    15:20我会给你新的分析报告。"
```

---

### 场景2: 新功能想法

```
你: "能不能加一个'每日冥想'功能？"

我: "很棒的想法！这个与我们'心理成长伙伴'的定位很契合。
    下次迭代我会：
    1. 设计'每日冥想'的具体功能
    2. 评估开发成本
    3. 分析对用户留存的帮助
    4. 看是否放入MVP还是V1.0
    
    你觉得这个功能应该免费还是付费？"
```
